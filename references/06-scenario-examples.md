### 案例1：新手第一次用 WorkBuddy

**用户原话**：「我刚装好 WorkBuddy，完全不知道从哪开始」

**对应卡片**：卡片7（新手引导）🔴 新手必读

**处理流程**：

```
Step 1: 立规矩
  问 Q1 → "我该叫你什么？"
  用户回答 → 记下称呼
  问 Q2 → "你主要是干什么工作的？"
  ... (逐问 5 个问题)

Step 2: 生成 Soul.md
  基于 5 个回答 → 按六条规矩书模板生成 → 展示给用户
  问 "OK 还是改？" → 用户确认 → 写入 ~/.workbuddy/SOUL.md

Step 3: 推荐技能
  "你的情况适合先装这 6 个通用技能：
   1. Skill安全扫描 2. Agent Browser 3. AI绘图
   4. PDF生成 5. Word生成 6. PPT生成
   要现在安装吗？"
```

**预期输出**：
- `~/.workbuddy/SOUL.md` 已写入
- 6 个通用技能安装成功
- 用户完成一次简单任务（如"帮我查今天天气"）

### 案例5：AI 回答不靠谱

**用户原话**：「让它分析数据，结果写了一堆废话，关键信息都没有」

**对应卡片**：卡片4（回答质量优化）🟡 进阶

Step 3: 重新提问 → AI 返回结构化分析
```

**预期输出**：
- 输出包含标题、表格、具体建议
- 格式一致，不再废话
- 连续 3 次提问都符合预期

### 案例7：搭一个分析报告专家团

**用户原话**：「我要写一份行业深度报告，能不能让多个 AI 角色一起协作？」

**对应卡片**：卡片6（多智能体）🟢 高级

**处理流程**：

```
Step 1: 定义专家团
  领队：research-lead（统筹分配）
  成员1：data-collector（收集行业数据）
  成员2：analyst（分析趋势和竞争格局）
  成员3：writer（基于分析撰写报告）

Step 2: 编写 SOP
  "1. 领队接收用户主题 → 拆解为数据收集/趋势分析/报告撰写
   2. 分配给 data-collector(收集行业数据)
   3. 分配给 analyst(分析趋势和竞争格局)
   4. 领队收集整合 → 分配给 writer
   5. writer 返回初稿 → 领队审核 → 输出最终报告"

Step 3: 配置 plugin.json
  expertType: "team"
  memberAgents: ["data-collector", "analyst", "writer"]

Step 4: 为每个成员编写 prompt（明确输入→处理→输出→停止条件）
```

**预期输出**：
- 团队创建成功，领队可调度
- 一次完整协作：输入主题 → 输出含数据+分析+撰写的报告
- 每个成员只执行自己的任务，不互相覆盖

### 案例8：全局诊断（问题不确定）

```

    □ 连接器状态（逐个）
    □ 专家列表（有无损坏）
    □ 日志文件（最近错误）

Step 2: 根据诊断结果处理
  示例输出：
    [OK] mcp_bridge running

Step 3: 修复后重新诊断
  → 全绿通过 ✅
```

**预期输出**：
- `check_env.py` 输出清晰的 OK/WARN/ERROR 分级
- 每个 WARN/ERROR 附带修复建议
- 修复后重新诊断全绿

### 案例10：运营 — 每日数据看板自动化

**用户原话**：「我每天早上 9 点需要看昨天的运营数据汇总，能不能自动生成？」

```

  rrule: "FREQ=DAILY;BYHOUR=9;BYMINUTE=0"

  prompt: "汇总昨天的运营数据，输出格式：
    ## 核心指标（列表：DAU/新增/留存率/转化率，附环比变化）

    数据来源优先使用内部数据工具，不可用则标注'数据待接入'"

Step 2: 验证首次执行
  手动触发一次 → 确认输出格式正确
  → 调整 prompt 直到满意

Step 3: 开启自动执行
  每天 9:00 自动生成 → 推送到飞书群或邮件
```

**预期输出**：
- 每天 9:00 自动生成结构化运营日报
- 异常自动标注预警
- 无需手动操作

### 案例16：财务 — 报表分析与异常检测

**用户原话**：「上个月的费用报表，帮我看看有没有异常支出，做个简要分析」

**对应卡片**：卡片5（金融数据查询）+ 卡片4（分析提示词优化）

**处理流程**：

```
Step 1: 明确分析维度
  「请分析这份费用报表，重点关注：
   ## 异常检测
      - 单项费用环比增长 > 30% 的条目
      - 预算执行率 < 50% 或 > 120% 的项目
      - 费用类别中占比异常的（某项超过总额 40%）
   ## 趋势分析
      - 近 3 个月费用变化趋势（表格）
      - 与去年同期的同比对比
   ## 改进建议
      - 标注 3 项最需要关注的费用
      - 每项给出优化建议和预期节省金额
   以 Markdown 格式输出，每部分不超过 200 字。」

Step 2: 如涉及外部数据
  需行业对标 → 用 neodata-financial-search 查行业平均费用率
  需汇率换算 → 确保展示人民币本位

Step 3: 进阶 — 月度自动分析
  建一个自动化任务，每月 1 号自动分析上月费用
  → 异常项推送到飞书
```

**预期输出**：
- 异常费用标注（超预算/环比异常）
- 趋势表格 + 同比对比
- 3 条可执行的优化建议

### 案例17：数据清洗 — 一句话搞定脏数据 🆕

**处理流程**：

```
Step 1: 一句话指令

   - 空值用列均值填充
   - 生成一张数据分布图
   保存为新文件到桌面"

- 清洗后的数据文件（无重复、无空值）
