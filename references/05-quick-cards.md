## 卡片1：MCP工具调用失败 🔴 新手必读

> 💨 急着修？[直接看修复表](#常见错误快速修复)

**症状**：调用工具时返回错误、无响应、或显示连接器disconnected

**快速诊断（3分钟）：**

```
[1] 确定问题范围
    是单个连接器还是全部？
    ├── 全部 → 代理可能挂了 → 重启 WorkBuddy
    └── 单个 → 继续 ▶

[2] 查看错误码
    ├── 401 → token过期 → 通过UI重新登录
    ├── 404 → URL路径错 → 检查配置
    ├── 超时 → 网络问题 → curl测试连通性
    └── method not found → tool名大小写错

[3] curl验证
    curl -s -o /dev/null -w "%{http_code}" https://api.xxx/mcp
    ├── 200 → 服务正常 → 检查认证头
    └── 非200 → 服务不可用
```

**验证修复**：
```
□ 重新调用同一个工具 → 返回预期结果（如 {"result": "success"}）
□ WorkBuddy设置中 → 该连接器显示 "connected"
□ 24小时内无再次失败 → 问题已修复
```

### 常见错误快速修复

| 错误提示 | 可能原因 | 修复操作 |
|---------|---------|---------|
| `401 Unauthorized` | Token 过期或未配置 | WorkBuddy UI → 连接器 → 重新登录授权 |
| `404 Not Found` | API 地址或路径错误 | 检查 plugin.json 的 `url` 字段，确认路径拼写 |
| `connection refused` | 代理未启动或端口冲突 | 重启 WorkBuddy；或 `netstat -ano \| findstr <port>` 查端口占用 |
| `timeout` / 请求超时 | 网络不通或防火墙拦截 | `curl -I <api_url>` 测试连通性；检查代理/防火墙 |
| `method not found` | tool 名称大小写或拼写错误 | 核对 plugin.json 中 tool name 与实际调用是否一致 |
| `SSL/TLS error` | 证书过期或自签名证书 | 检查服务器证书有效期；开发环境可临时跳过验证 |
| OAuth 授权弹窗不弹出 | 浏览器拦截或网络问题 | 检查系统默认浏览器是否被占用；尝试在 WorkBuddy 内嵌弹窗中操作 |
| OAuth Token 过期后未自动刷新 | 刷新机制配置异常 | 断开连接器 → 重新登录授权；检查 token 有效期配置 |
| 连接器间歇性断开重连 | OAuth 主动刷新线程异常 | 重启 WorkBuddy；确认网络稳定；检查连接器状态页面重连次数 |
| 全部连接器同时断开 | 代理进程崩溃 | 重启 WorkBuddy 桌面应用 |
| 工具调用被黑名单拦截 | 触发工具调用循环检测（v4.24.2+） | 检查调用链是否形成死循环；拆分复杂任务为独立步骤 |

**详细手册**：见 `02-connector-troubleshooting.md` 和 `troubleshooting.md`

## 卡片2：创建/修改专家 🔴 新手必读

> 💨 急着修？[直接看修复表](#常见错误快速修复-1)

**症状**：需要创建新专家或修改现有专家配置

### 创建新专家（三步法）

```
Step 1: 初始化目录
  通过 WorkBuddy 专家管理系统或 skill-creator 技能创建专家目录结构

Step 2: 编辑核心文件
  - plugin.json: 填充元数据（tags固定3个，quickPrompts固定3个）
  - agents/<name>.md: 编写专家人设（500-800字最佳）
  - 参考 03-expert-workflow.md §6 的人设设计模式

Step 3: 校验+注册
  通过 WorkBuddy 专家管理工具执行校验和注册
```

### 修改已有专家

```
1. 定位目录 → ~/.workbuddy/plugins/marketplaces/my-experts/plugins/<name>/
2. 编辑可改字段 → displayName/profession/tags/quickPrompts/agent.md
3. 严禁修改 → name/agentName/目录名/MD文件名
4. 重新注册 → validate + register
5. 重启WorkBuddy → 清除缓存
```

### ⚠️修改后不生效排查

```
□ 执行了专家校验？（必须）→ 输出应显示 "Validation passed"
□ 执行了专家注册？（必须）→ 输出应显示 "Registration successful"
□ 没改不可改字段？→ name/agentName/目录名未变
□ plugin.json 是合法JSON？→ 运行 `python -m json.tool plugin.json` 校验
□ 重启了WorkBuddy？→ 完全退出再启动，非最小化
```

### 常见错误快速修复

| 错误提示 / 现象 | 可能原因 | 修复操作 |
|---------------|---------|---------|
| `Validation failed` | plugin.json 格式错误或缺少必填字段 | `python -m json.tool plugin.json` 检查 JSON 语法；对照模板补全字段 |
| `Registration failed` | 目录结构不符合规范 | 确认 agents/ 目录存在、MD 文件名与 agentName 一致 |
| 注册成功但列表中不显示 | 未重启 WorkBuddy | 完全退出并重启桌面应用（重要：不是最小化） |
| 改了 agent.md 不生效 | 只保存未重新注册 | 修改后必须重新执行 `validate + register` |
| `SyntaxError: JSON` | plugin.json 中有非法字符 | 用 JSON 校验工具检查；注意最后一项后不能有逗号 |
| `File not found` | 路径或文件名不匹配 | 确认目录名、agentName、MD 文件名三者一致 |

**详细手册**：见 `03-expert-workflow.md` 和 `troubleshooting.md`

## 卡片3：配置自动化任务 🟡 进阶

> 💨 急着修？[直接看修复表](#常见错误快速修复-2)

**症状**：需要定时执行某个任务，或自动化不执行

### 创建自动化

```
循环任务：
  scheduleType: "recurring"
  rrule: "FREQ=DAILY;BYHOUR=9;BYMINUTE=0"
  status: "ACTIVE"

一次性任务：
  scheduleType: "once"
  scheduledAt: "2026-05-30T15:00:00+08:00"
  status: "ACTIVE"
```

### 自动化不执行？5步排查

```
1. status = "PAUSED"？→ 改成 "ACTIVE"
2. 刚创建不到5分钟？→ 等调度器轮询
3. 过了 validUntil？→ 修改有效期
4. cwds（工作目录）存在？
5. prompt 是自给自足的？（不能依赖交互）
```

### 管理命令

```
查看全部：    {"mode": "list"}
查看单个：    {"mode": "view", "id": "<id>"}
暂停：        {"mode": "update", "id": "<id>", "status": "PAUSED"}
删除：        {"mode": "delete", "id": "<id>"}  ← 只能用这个！
```

### 常见错误快速修复

| 错误提示 / 现象 | 可能原因 | 修复操作 |
|---------------|---------|---------|
| 任务到时间不执行 | status 为 "PAUSED" | 改为 "ACTIVE"；或 "list" 确认当前状态 |
| 刚创建不执行 | 调度器轮询间隔约 5 分钟 | 等待 5-10 分钟后观察 |
| 过了某天后不再执行 | validUntil 已到期 | 修改或删除 validUntil 限制 |
| 执行了但没效果 | cwds（工作目录）不存在 | 检查 cwds 路径使用绝对路径，确认目录存在 |
| 执行报错 | prompt 依赖交互（如"询问用户"） | prompt 必须自给自足，不能含"确认""选择"等交互词 |
| RRULE 语法错误 | FREQ/BYHOUR 等关键字拼写错误 | 使用标准 RFC 5545 格式，如 `FREQ=DAILY;BYHOUR=9;BYMINUTE=0` |
| 误删了任务 | 用了 rm/sqlite3 等命令 | **只能用** `automation_update` 工具删除 |

**详细手册**：见 `04-automation-patterns.md`

## 卡片4：前端问题排查 🟡 进阶

> 💨 急着修？[直接看修复表](#常见错误快速修复-3)

**症状**：AI回复不符合预期、输出格式错误、工具调用异常

### 排查流程

```
需要优化回答质量？
        │
        ▼
    [第一步] 确认现象类型
        │
        ├── 回答质量差 ──────────────────┐
        ├── 格式错误（没表格/太长） ──────┤
        ├── 工具异常（调用失败） ────────┤
        └── 对话太长/上下文混乱 ──────────┘
        │
        ▼
    [第二步] 对症下药
        ├── 回答质量差 → 优化提示词（四要素检查）
        ├── 格式错误   → 明确指定输出格式
        ├── 工具异常   → 检查连接器（按卡片1）
        └── 上下文混乱 → 开新会话 / 切换专家
        │
        ▼
    [第三步] 验证修复
        □ 重新提问 → 回答符合预期？
        □ 连续3次 → 都正常？
        □ 工具调用 → 都成功？
```

### 优化提示词（四要素检查）
    □ 角色：够具体吗？（"数据分析师" vs "帮助分析数据"）
    □ 任务：够明确吗？（不是"写方案"，是"写一份面向XX部门的XX方案"）
    □ 格式：够清晰吗？（Markdown表格 / JSON / 纯文本？）
    □ 约束：够严格吗？（"不要XXX"、"不确定就说不知道"）

[3] 检查上下文
    - 对话太长？→ 开新会话
    - 选错专家？→ 切换合适的专家

[4] 检查连接器
    → WorkBuddy设置 → 连接器 → 状态是否connected
    → disconnected → 按照卡片1排查

**常见提示词改善对照**：

| 问题 | 原句 | 优化后 |
|------|------|-------|
| 太模糊 | "帮我分析一下" | "用Markdown表格分析，包含数据、趋势、3条建议" |
| 无格式 | "写个报告" | "写300字报告，结构：背景→分析→结论→行动" |
| 无约束 | "推荐一些书" | "推荐3本XX领域的书，附50字理由，不要超过3本" |
| 太啰嗦 | "详细解释每个细节" | "用200字以内概述，每条要点不超过2行" |
| 无示例 | "写一封邮件" | "写一封XX场景的邮件，参考格式：称呼→正文3段→落款" |
| 语言混淆 | "给我一个summary" | 明确语言："用中文写一个英文摘要"或"用英文写300字总结" |

### 两大提质捷径

**捷径1：给虾看例子（模仿优于想象）**
AI 更擅长模仿而非凭空创造。提供一个满意样本，效果远胜于文字描述十行。
```
❌ "写一篇公众号文章，要有深度"（太抽象）
✅ 直接扔一篇你满意的历史文章，说"照这篇文章的口吻、结构、长度来写"
```

**捷径2：正向激励（PUA 虾）**
通过赋予高级角色和设置高标准，影响 AI 的回答深度。
```
❌ "帮我分析这个财报"
✅ "你是一位在投行工作10年的财务分析师，以你见过最优秀的研究报告为标准来写。我要有洞察力的分析，不是数据的搬运。"
```

**三要素公式**：做什么 + 有什么 + 怎么样
```
"把 D:/会议纪要/0320产品评审.docx 里的会议纪要整理成一份清单，
包含：1）每个议题的结论 2）对应的责任人和截止日期 3）标记争议项。
用表格输出，不需要开场白。"
```

### 常见错误快速修复

| 错误提示 / 现象 | 可能原因 | 修复操作 |
|---------------|---------|---------|
| 回答太啰嗦/太长 | 未指定长度或格式约束 | 在提示词末尾加「限制在200字以内，用3个要点概括」 |
| 回答过于简略 | 任务描述不够具体 | 加「请详细说明，包含步骤、原因、示例」 |
| 输出格式不对（没表格） | 未指定输出格式 | 加「请用 Markdown 表格输出，列名为 XXX」 |
| AI 说「我不确定」过于频繁 | 约束词过度严格 | 改为「不确定就说明，但请先尝试给出最可能答案」 |
| 工具调用穿插在文本中 | 上下文混乱或专家切换了 | 开新会话，重新选择对应专家 |

**详细手册**：见 `02-connector-troubleshooting.md` 和 `03-expert-workflow.md`

## 卡片5：金融数据查询选择 🟢 高级

> 💨 急着修？[直接看修复表](#常见错误快速修复-4)

**症状**：需要查询股票/基金/宏观数据，不知道用哪个skill

### 决策树

```
需要查询金融数据？
        │
        ▼
    [第一步] 尝试 neodata-financial-search
    → 自然语言提问，A股/港股/美股，实时数据
    → 适合：行情、财报、基金、板块、宏观
    → 限制：公募基金主覆盖中国境内、板块主覆盖A股

    [第二步] neodata不满足？
    → 切换 westock-data（结构化数据）
    → 适合：技术指标、筹码、股东结构、龙虎榜、融资融券
    → 命令：westock-data <command> <code> [options]

    [第三步] 两个都不满足？
    → 使用 WebSearch 检索公开信息
    → 告知用户数据来源和非实时性
```

### westock-data 常用命令（可直接复制）

```bash
# 实时行情              → westock-data quote sh600519
# K线（最近20天）        → westock-data kline sh600519 --period day --limit 20
# 财务报表（最近4期）     → westock-data finance sh600519 --num 4
# A股资金流向            → westock-data asfund sh600519
# 技术指标(MACD)         → westock-data technical sh600519 --group macd
# 筹码成本（仅A股）       → westock-data chip sh600519
# 股东结构（A股/港股）     → westock-data shareholder sh600519
# 分红数据               → westock-data dividend sh600519
# ETF详情                → westock-data etf sh510300
# 搜索股票               → westock-data search 腾讯控股
# 投资日历               → westock-data calendar 2026-05-29
# 新股日历               → westock-data ipo hs
# 宏观经济(GDP)           → westock-data macro --indicator gdp --year 2025
```

**代码格式**：沪市 `sh600519` / 深市 `sz000001` / 港股 `hk00700` / 美股 `usAAPL`
**⚠️ 港股/美股展示时标注正确货币单位（港币/美元），不要用¥**

### 常见错误快速修复

| 错误提示 / 现象 | 可能原因 | 修复操作 |
|---------------|---------|---------|
| neodata 返回"未找到" | 查询词太模糊或不在覆盖范围 | 改用股票代码 + 具体指标，或切换 westock-data |
| westock 命令报错 | 代码格式不对 | 检查前缀：沪`sh`/深`sz`/港`hk`/美`us`，如 `sh600519` |
| 港股/美股显示人民币 | 未切换货币单位 | 展示数据时手动标注「港币 HKD」或「美元 USD」，不用 ¥ |
| `command not found: westock-data` | 技能未安装 | 在 SkillHub 搜索安装 westock-data 技能 |
| neodata 数据不是最新的 | 免费版可能有延迟 | 切换 westock-data；或注明数据时间戳 |
| 查不到某只股票 | 代码或市场不对 | 用 `westock-data search 股票名称` 先搜索确认代码 |

## 卡片6：多智能体工作流（专家团协作） 🟢 高级

> 💨 急着修？[直接看修复表](#常见错误快速修复-5)

**症状**：需要多个AI角色协作完成复杂任务

### Team型专家结构

```json
{
  "expertType": "team",
  "members": [
    {"role": "lead", "name": "team-lead", "title": "负责人"},
    {"role": "member", "name": "member-1", "title": "成员1"},
    {"role": "member", "name": "member-2", "title": "成员2"}
  ],
  "teamInfo": {
    "memberAgents": ["member-1", "member-2"],
    "sop": "1. 领队接收任务 → 2. 分配给member-1(XX任务) → 3. 分配给member-2(YY任务) → 4. 领队收集整合 → 5. 输出结果"
  }
}
```

### 协作流程

```
领队（Lead）
    ├── 接收用户任务 → 拆解为子任务
    ├── 分配给各成员（写明 receiver + action + output）
    ├── 收集成员输出 → 整合
    └── 输出最终结果

成员（Member）
    ├── 接收子任务（从prompt中得知如何接收）
    ├── 独立完成专业工作
    └── 返回结果给领队
```

### 常见失败原因

| 现象 | 原因 | 修复 |
|------|------|------|
| 成员不工作 | memberAgents与实际文件名不匹配 | 确保名称一致 |
| 领队不分配 | SOP太模糊，不知道分给谁 | 每步写明接收者 |
| 成员不会返回 | prompt没写返回格式 | 在成员prompt中加入"返回格式" |
| 结果不整合 | 领队没收到成员输出 | 检查memberAgents列表 |

### 典型场景配置

| 场景 | 专家团 | 分工 |
|------|--------|------|
| 深度研究报告 | researcher + analyst + writer | 数据收集→分析→撰写 |
| HR全流程 | jd-writer + resume-screener + interviewer | JD→筛选→面试 |
| 代码审查 | architect + reviewer + tester | 架构→代码→测试 |
| 人设打磨 | designer + optimizer + tester | 定义→优化→测试 |

### 常见错误快速修复

| 错误提示 / 现象 | 可能原因 | 修复操作 |
|---------------|---------|---------|
| 成员不工作（无任何输出） | memberAgents 列表与实际文件名不匹配 | 核对 plugin.json 中 memberAgents 与 agents/ 目录下文件名完全一致 |
| 领队不分配任务 | SOP 描述太模糊，未指定「谁」做什么 | SOP 中每步写明成员名称：`分配给 member-1(收集数据)` |
| 成员输出后领队不整合 | 领队 prompt 未写「等待所有成员完成后整合」 | 在领队 prompt 中明确等待逻辑和整合格式 |
| 多个成员输出互相覆盖 | 任务未区分成员职责边界 | 确保每个成员的 prompt 明确其独特任务范围 |
| 团队创建后选不中 | plugin.json 中 expertType 未设为 "team" | 修改 expertType 字段为 "team"，重新注册 |
| 成员反复执行同一步 | 成员 prompt 缺少停止条件 | 加「完成XX后停止，等待领队下一步指令」

**详细手册**：见 `03-expert-workflow.md`

## 卡片7：新手引导 / 立规矩（聊天式分步配置） 🔴 新手必读

> 💨 急着配置？[直接看 Soul.md 模板](#soulmd-六条规矩书模板可直接复用)

**症状**：新用户不知道怎么上手 WorkBuddy，或需要为 WorkBuddy 设定用户偏好规矩

### 三步框架（通用引导模板）

核心方法论源自 SOUL.md 四大支柱：**核心性格 + 说话方式 + 工作原则 + 红线**。

```
Step 1: 立规矩（互动采集流程，每次只问1个问题）
  启动："准备好了吗，让我来修改你的性格档案"
  Q1 → 你希望我说话什么风格？（命令式短句 / 分析式 / 轻松 / 严肃）
  Q2 → 有没有你特别讨厌的回答方式？（如"不要客套话""不要说好的呢"）
  Q3 → 工作中有什么原则是你特别看重的？（"拿不准先问""不要省略步骤"…）
  Q4 → 有没有绝对不能碰的红线？（内容类型/语气/合规边界）
  → 基于 4 大支柱生成六条规矩书 → 展示确认 → "OK 还是改？"
  → 确认后写入 ~/.workbuddy/SOUL.md，开启全局记忆

Step 2: 安装基础设施技能（按职业推荐）
  通用必装（6项）：
    1. Skill安全扫描（朱雀） 2. Agent Browser
    3. AI绘图              4. PDF文档生成
    5. Word文档生成         6. PPT演示文稿
  按职业额外推荐：
    自媒体/内容运营 → 小红书图文发布、微信公众号发布、AI写作助手
    企业培训         → AI视频生成、Excel文件处理
    开发/技术        → GitHub热门项目、ArXiv论文追踪

Step 3: 下指令练习（可选）
  从低难度任务开始实操，熟悉工具调用
  后续维护：随时用"你要…""你应该…""不要…"微调规则，无需重新走完整流程
```

### Soul.md 六条规矩书模板（可直接复用）

```
第一条：核心性格        → 用 2-3 句话定义你是谁（身份/职业/核心价值），性格兜底
第二条：说话方式        → 该说什么、不该说什么。消除 AI 味：禁"好的呢""您说得太对了"
第三条：工作原则        → 3-5 条铁律：带方案来、干完复盘、一次到位、不确定就说
第四条：嘴严            → 隐私/文件/聊天记录/未公开内容，绝不泄露不转述
第五条：红线禁区        → 法务合规底线 + 个人雷区（特定话题/语气/内容），绝不越界
第六条：持续进化        → AI 出主意、用户做决定；每次互动都是学习机会，用"你应该…""不要…"随时修正
```

### 聊天式引导规范（适用任何配置向导）

- ✅ 每次只问 **1个** 问题，不给表格让用户填
- ✅ 等用户回答后再问下一个（不要一次性抛多个问题）
- ✅ 生成结果后只问 **"OK 还是改？"**（简洁确认）
- ✅ 用户确认后 **自动写入** 配置文件，不要再次询问
- ❌ 不要跳过任何步骤（每个问题都要等用户回答）
- ❌ 不要语气官方套话（像朋友聊天）

### 新手常见误区（来自真实用户踩坑）

| 误区 | 反面示范 | 正确做法 |
|------|---------|---------|
| 需求模糊 | "帮我把上次的会议纪要整理一下" | 使用三要素公式：**做什么 + 有什么 + 怎么样**（例："把 D:/会议纪要/0320.docx 整理成清单，表格输出，含结论/责任人/争议事项"） |
| 任务过大 | "读完50页报告，直接写PPT发领导" | 拆成3轮：①提炼5条核心结论 → ②生成大纲 → ③逐页扩展。一次只推进一步 |
| 轻易放弃 | 第一次回答不满意就放弃 | 多轮打磨："太正式了→换成轻松语气→200字以内→结尾加一句XX"。AI 像实习生，越具体越靠谱 |
| 死磕上下文 | 一个会话聊了50轮还继续追加 | 不同任务开独立会话；过长对话果断开新窗口，简洁重述背景 |
| 不备份直接改 | 让 AI 直接修改原始重要文件 | 修改前先复制备份（加 `_备份` 或日期后缀），出错可随时回滚 |
| 只用聊天框 | 打开→输入→等答案→复制→关闭，不用任何功能模块 | WorkBuddy 不是聊天机器人，是"同事"。善用左侧专家/自动化/探索/Skills 模块 |
| 不分工作空间 | 所有项目混在一个空间，上下文互相污染 | **输入前**先在右下角选好/创建对应工作空间，每个项目独立存放 |

### 10大核心技巧速览

| # | 技巧 | 一句话 | 对应卡片 |
|---|------|--------|---------|
| 1 | 清晰表达 | 指令越清晰，结果越精准：**做什么 + 有什么 + 怎么样** | 卡片4 |
| 2 | 小步快跑 | 大任务拆成3-5个小步骤，一次只推进一轮 | 卡片4 |
| 3 | 反复打磨 | 同一个需求换个说法再问，AI 对措辞非常敏感 | 卡片4 |
| 4 | 先本地后远程 | 先在本地桌面端练熟，再开远程/自动化 | 卡片3 |
| 5 | 给虾身份 | 用「专家」功能设定角色，回答框架更对口 | 卡片2 |
| 6 | 给虾看例子 | 扔一个满意样本，说"照这个来"，比文字描述快十倍 | 卡片4 |
| 7 | 任务隔离 | 不同任务开不同会话，保持上下文干净 | 卡片4 |
| 8 | 勤备份 | 改任何重要文件前，先复制一份副本 | 卡片4 |
| 9 | 自动化 | 重复性任务设成定时自动化，24小时后台运转 | 卡片3 |
| 10 | 正向激励 | "你是顶尖XX师，以最好标准来"，影响回答深度和框架 | 卡片4 |

### 常见错误快速修复

| 错误提示 / 现象 | 可能原因 | 修复操作 |
|---------------|---------|---------|
| Soul.md 写入后不生效 | WorkBuddy 未重启或缓存未刷新 | 完全退出 WorkBuddy 再重新打开 |
| 引导中途卡住 | 一次性问了多个问题 | 严格遵守：每次只问 1 个问题，等回答 |
| 生成的规矩书用户不满意 | 未逐条确认就写入 | 生成后先展示 → 问「OK 还是改？」→ 确认后再写入 |
| 聊天语气太官方 | 未采用聊天式引导 | 用「我该叫你什么？」而非「请输入您的称呼」 |
| 技能安装后看不到 | 技能市场同步延迟 | 退出再进入技能列表；或手动刷新技能市场 |
| 不知道该装哪些技能 | 职业推荐不匹配 | 参考卡片7 Step 2 的职业分类，优先装通用必装 6 项 |
