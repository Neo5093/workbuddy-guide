# 08 — 术语表（Glossary）

> **版本**：v1.1 (2026-06-04) — 新增「模型」和「网络安全」两章
> **用途**：快速查找 WorkBuddy 生态中的核心概念定义

---

## WorkBuddy 核心

| 术语 | 全称/英文 | 一句话定义 |
|------|----------|-----------|
| **WorkBuddy** | — | 能实际执行任务的 AI 助手平台，可操作本地文件、调用外部服务、定时自动化运行 |
| **连接器 (Connector)** | Connector / MCP | WorkBuddy 与外部服务之间的桥梁，如 GitHub、飞书、乐享等 |
| **MCP** | Model Context Protocol | 连接器的底层协议标准，定义了工具调用的格式和认证方式 |
| **代理 (Bridge/MCP Bridge)** | — | WorkBuddy 桌面端运行的本地进程，负责转发所有 MCP 工具调用 |
| **SSE** | Server-Sent Events | MCP 连接器与 WorkBuddy 通信的协议，支持实时数据推送 |

---

## 专家系统

| 术语 | 全称/英文 | 一句话定义 |
|------|----------|-----------|
| **专家 (Expert)** | Expert / Agent | 预设了角色、知识和工作方式的 AI 对话模式，如"法律顾问""数据分析师" |
| **专家团 (Expert Team)** | Team | 多个专家角色协作的团队，由领队分配任务、成员各司其职 |
| **plugin.json** | — | 专家/团队的配置文件，定义名称、标签、快捷指令、成员列表等元数据 |
| **SOP** | Standard Operating Procedure | 专家团的工作流程定义：谁、做什么、什么顺序、输出什么 |
| **专家管理系统** | expert-manager | WorkBuddy 内置的专家创建、修改、校验、注册工具链 |

---

## 技能与能力

| 术语 | 全称/英文 | 一句话定义 |
|------|----------|-----------|
| **技能 (Skill)** | Skill | 封装了 SOP + 工具 + 脚本的能力扩展包，安装后获得新功能（如 PPT 生成、PDF 处理） |
| **技能市场 (SkillHub)** | SkillHub | WorkBuddy 的技能商店，浏览、安装、发布技能 |
| **Skill vs 专家** | — | 技能 = 扩展能力（固定格式输出）；专家 = 对话模式（聊背景聊需求） |

---

## 自动化

| 术语 | 全称/英文 | 一句话定义 |
|------|----------|-----------|
| **自动化 (Automation)** | Automation | 定时自动执行的任务，如"每天早上 9 点推送新闻" |
| **RRULE** | Recurrence Rule (RFC 5545) | 定时任务的重复规则语法，如 `FREQ=DAILY;BYHOUR=9;BYMINUTE=0` |
| **调度器** | Scheduler | WorkBuddy 后台轮询自动化的组件，约每 5 分钟检查一次待执行任务 |

## 知识管理

| 术语 | 全称/英文 | 一句话定义 |
|------|----------|-----------|
| **IMA** | Intelligent Memory Assistant | 腾讯个人知识管理平台，支持资料存储、知识库问答和跨产品调用 |
| **Soul.md** | — | WorkBuddy 的性格配置文件，定义 AI 的说话风格、工作原则、红线禁区 |
| **工作空间 (Workspace)** | Workspace | WorkBuddy 中隔离不同项目的独立空间，每个空间有独立的对话和文件上下文 |

## 远程与互通

| 术语 | 全称/英文 | 一句话定义 |
|------|----------|-----------|
| **Claw** | — | WorkBuddy 的远程通道，可接入微信/QQ/企业微信，手机发指令、电脑在线执行 |
| **WebFetch** | — | WorkBuddy 内置的网页内容抓取工具，用于检索官方文档等在线资源 |

## 配置与文件

| 术语 | 全称/英文 | 一句话定义 |
|------|----------|-----------|
| **~/.workbuddy** | — | WorkBuddy 的用户配置根目录，包含 SOUL.md、skills、plugins 等 |
| **session-id** | — | WorkBuddy 会话的唯一标识，注册专家时需要引用当前会话 |
| **Tenant / 租户** | Tenant | WorkBuddy 中隔离不同企业/组织的逻辑分区，token 和 company_from 必须属于同一租户 |
| **产物分享** | Artifact Sharing | 将 WorkBuddy 生成的报告/PPT/图表等产物通过微信二维码或 HTML 链接分享给他人（v4.22.14+） |
| **OAuth 应用** | OAuth Application | WorkBuddy 支持的标准授权协议，用于连接外部服务时的安全认证和 token 管理（v4.7.3+） |
| **computer-use** | — | WorkBuddy 实验性功能（v4.22.4），允许 AI 直接操控计算机桌面，目前仅支持 macOS |
| **工具调用循环检测** | Tool Loop Detection | v4.24.2 新增的安全机制，自动检测并拦截 AI 工具调用的死循环 |

## 模型（Models）

| 术语 | 全称/英文 | 一句话定义 |
|------|----------|-----------|
| **Auto 模型** | — | WorkBuddy 智能调度，按任务类型自动分配最合适的模型，平衡效果与成本 |
| **模型倍率 (Credit)** | Credit Multiplier | 模型消耗积分的倍率系数，如 0.40x = 基础消耗的 40%，用于对比模型性价比 |
| **Thinking Mode** | Reasoning Mode | 模型思维链展示模式，适合复杂推理，开启后模型展示推理过程但消耗更多积分 |
| **ModelId** | Model ID | 模型标识符，在自动化和专家配置中指定使用哪个底层模型 |
| **Hy3 preview** | — | 腾讯混元第三代预览版，倍率极低（0.04x），适合批量处理、简单分类、轻量改写 |
| **GLM 系列** | 智谱 AI | 中文综合能力强的模型系列，GLM-5.1 为当前最强中文模型 |
| **Kimi 系列** | 月之暗面 | 长上下文处理专长模型，Kimi-K2.6 适合超长文档分析与摘要 |
| **Deepseek 系列** | 深度求索 | 编程能力突出的模型系列，V4-Pro 为编程任务首选 |
| **MiniMax M2.7** | — | 性价比黑马，中文表现稳定，在 Auto 与 GLM 之间的折中选择 |

### 模型选择速查

| 场景 | 推荐模型 | 倍率 |
|------|---------|------|
| 不确定选什么 | Auto | 0.40x |
| 批量/轻量任务 | Hy3 preview | 0.04x |
| 中文写作/复杂推理 | GLM-5.1 | 1.06x |
| 编程任务 | Deepseek V4-Pro | 0.25x |
| 长文档分析 | Kimi-K2.6 | 0.59x |
| 看图理解 | GLM-5v-Turbo | 0.95x |
| 性价比折中 | MiniMax M2.7 | 0.26x |

## 网络安全（Cyber Safety）

| 术语 | 全称/英文 | 一句话定义 |
|------|----------|-----------|
| **Bash 沙箱** | Bash Sandbox | 文件系统隔离层，AI 执行的所有命令被限制在明确允许的工作空间内 |
| **沙箱逃逸** | Sandbox Escape | 攻击者试图突破沙箱限制访问系统敏感路径的攻击方式，WorkBuddy 内置防逃逸机制 |
| **纵深防御** | Defense in Depth | 三层安全架构：技术防御（沙箱）+ 管理管控（企业后台）+ 合规审计（加密追溯） |
| **SSO** | Single Sign-On | 企业统一身份认证，组织架构自动同步，管理员统一管控用户权限 |
| **Credit 额度控制** | Credit Quota | 按部门/角色差异化设置 AI 使用额度，防止资源滥用 |
| **IP 白名单** | IP Whitelist | 限制只有特定 IP 段才能登录，企业内网安全策略 |
| **VPC 专享版** | VPC Dedicated | 面向金融/政务等强监管行业的物理隔离部署方式，数据绝对主权 |
| **提示注入** | Prompt Injection | 恶意内容伪装成合法指令，诱导 AI 执行非预期操作。Bash 沙箱 + 权限确认双重拦截 |
| **Skill 安全审计** | Skill Security Audit | 社区/第三方技能包由腾讯安全团队联动做分级风险判别（P0/P1/P2） |
| **数据本地执行** | Local Execution | 数据在用户本地处理不上传云端，从源头杜绝泄漏 |
| **工具调用权限** | Tool Permission | 像配防火墙一样为 AI 设定文件/命令/网络的允许/询问/拒绝策略 |
