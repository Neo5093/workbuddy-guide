# 11 — 记忆系统详解（Memory System）

> **版本**：v1.0 (2026-06-04)
> **用途**：理解 WorkBuddy 的三层记忆架构，让 AI 跨会话记住有用上下文
> **来源**：WorkBuddy 官方教程 — Memories 章节 + 系统 Prompt 记忆规范

---

## 三层记忆架构

WorkBuddy 的记忆不是单一系统，而是按"作用范围"分了三层：

```
层1  Cloud Memory（云记忆·服务端）
      自动生成的长期用户画像，跨所有项目生效
         │
层2  User-level Local Memory（用户记忆·本地文件）
      ~/.workbuddy/MEMORY.md — 跨项目的手动规则
         │
层3  Workspace Memory（工作区记忆·本地文件）
      .workbuddy/memory/ — 项目级别的日志与长期笔记
```

---

## 层1：Cloud Memory（云记忆）

### 是什么
- WorkBuddy 服务端自动总结的长期用户画像
- 每次新会话开始时，画像被自动注入到对话开头（`<memory>` 块）
- 你感知不到写入过程，无需手动记录

### 画像内容

注入到对话开头的 `<memory>` 块通常包含：

| 分类 | 内容示例 |
|------|---------|
| **工作背景** | 职位、团队、负责产品、推进中的项目 |
| **个人背景** | 身份、语言偏好、做事风格 |
| **当前关注** | 最近反复出现的话题、正在迭代的工作、近期目标 |
| **近期动态** | 过去若干次会话的关键决策、新落地成果、新偏好 |

### 画像怎么积累

| 机制 | 说明 |
|------|------|
| **自动总结** | 服务端自动总结多次会话中的稳定信息 |
| **越聊越准** | 随口信息不立刻写入，反复出现或明确强调的偏好会沉淀 |
| **持续更新** | 新工作进展、新约定会替换或补充旧条目 |
| **主动加速** | 说「以后都按 XXX 来」，让画像更快收敛 |

### 如何管理

- **默认开启**，开箱即用
- 设置 → 记忆中可管理：
  - 总开关：可一键关闭，关闭后新会话不再注入画像
  - 重新开启：随时可再打开，服务端继续积累
- ⚠️ 关闭只影响画像注入，不影响本次对话本身的上下文

### 适合 / 不适合记什么

| ✅ 适合 | ❌ 不适合 |
|---------|----------|
| 长期身份信息 | 一次性临时信息 |
| 稳定偏好（分点/表格/极简） | 敏感数据（密码/身份证） |
| 长期项目背景与角色 | 频繁变动的状态 |
| 反复确认的禁用词与约定 | 只在某项目里有意义的细节 |

### 让画像更好用的技巧

1. **明确表达偏好**："以后给我的文档都用表格对比"——比模糊描述更易沉淀
2. **必要时主动澄清**：发现过时或错误条目，告诉 AI 更正
3. **关键决策重复强调**：重要约定多说几次，确保进入长期画像
4. **隐私优先**：涉及敏感内容时，可临时关闭 Memories 开关再对话

## 层2：User-level Local Memory（用户本地记忆）

### 是什么
- 文件路径：`~/.workbuddy/MEMORY.md`
- 作用范围：**跨所有项目生效**
- 写入方式：**你主动要求 AI 记住时**，AI 写入此文件

### 适合记什么
- 跨项目的个人偏好（如"所有报告用 Markdown"）
- 强制规则（如"绝对不要用 emoji"）
- 不被 Cloud Memory 覆盖的特定指令

### 与 Cloud Memory 的区别

| 对比 | Cloud Memory | Local Memory |
|------|-------------|--------------|
| 写入方式 | 自动（隐式） | 手动（显式要求 AI） |
| 更新频率 | 持续 | 按需 |
| 内容 | 画像与偏好摘要 | 精确的强制性规则 |
| 可控性 | 低（自动生成） | 高（你编辑） |

---

## 层3：Workspace Memory（工作区记忆）

### 是什么
- 目录：`{workspace}/.workbuddy/memory/`
- 作用范围：**仅当前项目**
- 包含两类文件：

| 文件 | 用途 | 写入规则 |
|------|------|---------|
| `YYYY-MM-DD.md` | 每日工作日志 | 追加模式（append-only），不可覆写 |
| `MEMORY.md` | 长期项目笔记 | 按主题整理和更新 |

### 每日日志（YYYY-MM-DD.md）
- AI 完成实质性工作后自动追加一条记录
- 记录内容：做了什么、选择了什么方案、bug 修复等
- 不记录：临时搜索结果、临时路径、工具错误

### 长期笔记（MEMORY.md）
- 从超过 30 天的每日日志中提炼
- 按主题聚合（如"架构决策""踩坑经验""配置规范"）
- 项目约定和偏好统一放这里

## 一句话总结

```
Cloud 管画像 ──→ 自动注入，让 AI 知道你是谁
Local 管规矩 ──→ 精确规则，跨项目强制执行
Workspace 管项目 ──→ 项目级别的上下文积累
```

## 记忆系统的常见问题

### Q: AI 好像记不住我的偏好怎么办？
→ 用明确句式表达：「以后都按 XXX 来」——比模糊描述更容易进入 Cloud Memory。

### Q: 某些偏好只在这个项目需要，怎么办？
→ 让 AI 写入 Workspace MEMORY.md，不污染全局记忆。

### Q: 担心隐私泄露？
→ 敏感对话前，临时关闭 Memories 开关（设置 → 记忆 → 总开关）。关闭后本次对话不会更新画像。

### Q: Cloud Memory 的画像有错误？
→ 直接告诉 AI：「我现在的角色变了，是 XXX 而不是 YYY」，AI 会逐步修订。
