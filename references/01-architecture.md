> **版本**：v1.1 (2026-06-04) — 新增「六层定制化体系」章节
> **用途**：建立 WorkBuddy 系统架构的"心理地图"，帮助快速理解组件关系

---

## 一、WorkBuddy 是什么？（一句话定义）

> **WorkBuddy = 能实际执行任务的 AI 助手平台**  
> 区别于传统聊天AI，它可以：操作本地文件、调用外部服务、执行多步骤工作流、定时自动运行

## 三、五大核心概念

### 3.1 连接器 (Connector)

| 维度 | 说明 |
|------|------|
| **定义** | WorkBuddy 与外部服务的桥梁，基于 MCP 协议实现数据互通 |
| **类比** | 遥控器 —— 让 AI 能帮你操作其他软件/服务 |
| **核心能力** | 数据查询、服务调用、文件管理、消息通知 |
| **配置方式** | 左侧导航栏「连接器」→ 点击「+」→ 扫码/授权/断开 |
| **典型配置** | 通用配置示例：可连接多个常用服务（如网盘、邮箱、文档、IM等） |
| **未连接** | 其他服务（按需启用） |

### 3.2 技能 (Skill)

| 维度 | 说明 |
|------|------|
| **定义** | 可复用的能力组件，扩展 WorkBuddy 的功能边界 |
| **类比** | 插件/扩展包 —— 装上就能用新功能 |
| **来源** | 官方预置 / 技能市场安装 / 用户自定义创建 |
| **结构** | `SKILL.md`（说明）+ `references/`（参考资料）+ `scripts/`（执行脚本） |
| **管理方式** | 技能市场 → 添加技能 → 上传/查找/创建 |

### 3.3 专家 (Expert)

| 维度 | 说明 |
|------|------|
| **定义** | 具备特定领域知识和人设的 AI 角色 |
| **类型** | 单专家（Agent型）/ 专家团（Team型，多角色协作） |
| **类比** | 召唤对应岗位的资深从业者 / 项目组协同 |
| **示例用法** | my-expert（自定义专家示例） |

### 3.4 自动化 (Automation)

| 维度 | 说明 |
|------|------|
| **定义** | 定时/周期性自动执行的任务 |
| **类比** | 闹钟 + 机器人 —— 到点自动干活 |
| **配置项** | 名称、工作空间、提示词、模型/技能、定时规则、推送开关 |
| **调度类型** | 一次性（once）/ 周期性（recurring，支持 cron 规则） |
| **有效期** | 可设置 validFrom / validUntil |

### 3.5 MCP 协议

| 维度 | 说明 |
|------|------|
| **全称** | Model Context Protocol |
| **作用** | 标准化 AI 与外部工具的通信协议 |
| **关键点** | 连接器基于 MCP 协议实现，WorkBuddy 负责协议转换和消息路由 |

## 六、六层定制化体系（Customization Hierarchy）

WorkBuddy 的定制化不是靠单一设置完成，而是六层能力配合。从底层到顶层逐步增强：

```
层6  Expert Teams（专家团）  ← 多角色跨领域协作
层5  Experts（专家）         ← 专业任务交给专业角色
层4  Connectors（连接器）    ← 接入外部工具与系统
层3  Automations（自动化）   ← 周期性事务自动跑
层2  Skills（技能）          ← 可复用工作流与领域经验
层1  Memories（记忆）        ← 沉淀画像与习惯，自动注入
```

### 各层详解

| 层级 | 名称 | 一句话作用 | 触发方式 | 配置难度 |
|------|------|-----------|---------|---------|
| 1 | **Memories** | 自动记住你是谁、喜欢什么 | 自动沉淀，无需手动 | ⭐ 零配置 |
| 2 | **Skills** | 装了就能用新功能 | 命中任务时按需加载 | ⭐⭐ 安装/创建 |
| 3 | **Automations** | 到点自动干活 | 定时触发 | ⭐⭐ 配置规则 |
| 4 | **Connectors** | 让 AI 操作外部服务 | 按需调用 | ⭐⭐⭐ 授权配置 |
| 5 | **Experts** | 专业角色对话 | 手动切换/对话中召唤 | ⭐⭐⭐ 自定义创建 |
| 6 | **Expert Teams** | 多专家协作出活 | 团队模式下协同 | ⭐⭐⭐⭐ 定义 SOP |

### 定制化路径推荐

| 你的状态 | 推荐路径 |
|---------|---------|
| **第一天用** | 先走 1→4：配置 Memories + 连一个 Connector，就能日常用了 |
| **一周后** | 追加 2→5：装几个 Skills + 创建 1-2 个常用专家 |
| **深度用户** | 补上 3→6：设 Automation 自动跑 + 搭 Expert Team 多角色协作 |

> 不必一次全配齐。每加一层都会明显提升体验，按需逐步叠加即可。

## 八、常见误区澄清

| 误区 | 正解 |
|------|------|
| "连接器 = 技能" | ❌ 连接器是"接入外部服务的能力"，技能是"完成特定任务的封装" |
| "专家 = 技能" | ❌ 专家是人设+方法论+工具链，技能是工具能力 |
| "自动化 = 手动任务" | ❌ 自动化是定时调度，手动任务是即时触发 |
| "MCP = 连接器" | ❌ MCP是协议标准，连接器是基于该协议的实现 |
